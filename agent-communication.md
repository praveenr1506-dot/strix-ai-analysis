## Agent Communication Flow in Strix

Strix uses a multi-agent architecture where different agents collaborate to perform autonomous penetration testing. Each agent is responsible for a specific stage of the scanning workflow and communicates by sharing structured context and tool outputs.

The process begins with the Recon Agent, which explores the target application by crawling pages, discovering endpoints, and identifying input parameters. The information collected during this phase is passed to the Mapping Agent.

The Mapping Agent organizes the discovered data into a structured attack surface representation such as a sitemap or endpoint graph. This helps other agents understand which areas of the application require deeper testing.

Next, the Vulnerability Agent uses this contextual information to perform targeted security checks. It runs automated scanning tools, injects payloads, and evaluates responses to identify potential vulnerabilities.

If suspicious behavior is detected, the Exploitation Agent attempts to validate the finding by reproducing the attack and confirming whether it can be exploited. This reduces false positives and ensures that only meaningful issues are reported.

Finally, the Reporting Agent aggregates validated findings and generates structured output such as vulnerability summaries, severity levels, and remediation suggestions.

Agent communication is coordinated through shared memory, intermediate scan results, and reasoning decisions generated by the language model. This dynamic interaction enables adaptive and stateful penetration testing workflows.
